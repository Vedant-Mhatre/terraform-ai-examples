# Distribution Checklist

## Before Posting

- [ ] README top section clearly communicates value in 5 seconds.
- [ ] One screenshot/diagram prepared.
- [ ] One concrete calculation example prepared (from `System Design Sizing`).
- [ ] Repo passes `make validate`.

## Posting Cadence

- [ ] Week 1: Launch post (LinkedIn + X + Reddit).
- [ ] Week 2: Share one incident runbook walkthrough.
- [ ] Week 3: Share one cost-sizing example.
- [ ] Week 4: Ship release notes + changelog summary.

## Conversion Optimizations

- [ ] Include direct star CTA in posts.
- [ ] Link directly to one strong example, not just repo root.
- [ ] Add one image/diagram in each announcement.

## Metrics To Track

- [ ] Stars/day
- [ ] Traffic source to repo (GitHub insights)
- [ ] Top-viewed files
- [ ] Conversion after each post
